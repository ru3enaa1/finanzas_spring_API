Gestión de usuarios

RF1. El sistema debe permitir registrar usuarios con nombre, apellido, correo y contraseña.

RF2. El sistema debe validar el inicio de sesión con correo y contraseña.

RF3. El sistema debe permitir a los usuarios editar su información personal.

Gestión de cuentas

RF4. El sistema debe permitir al usuario registrar una o varias cuentas (ejemplo: ahorro, corriente).

RF5. El sistema debe almacenar y actualizar el saldo de cada cuenta según ingresos y gastos.

Gastos fijos

RF6. El sistema debe permitir registrar gastos fijos mensuales (ejemplo: arriendo, servicios, préstamos).

RF7. El usuario podrá ingresar un monto estimado para cada gasto fijo.

RF8. El sistema debe permitir marcar mensualmente si el gasto fijo fue pagado, y registrar el monto pagado real (tabla registro_gasto_fijo).

RF9. Si el gasto fijo no se paga, el sistema debe generar una alerta automática.

Ingresos y gastos variables (transacciones)

RF10. El sistema debe permitir registrar ingresos variables (ejemplo: lotería, inversiones).

RF11. El sistema debe permitir registrar gastos variables (ejemplo: cine, restaurantes).

RF12. El usuario podrá asociar cada transacción a una cuenta específica.

RF13. El sistema debe recalcular el saldo de la cuenta cada vez que se registre una transacción.
